## Context

`pnpm audit` reports 2 moderate vulnerabilities after upgrading `@fastify/static` to `^9.1.1`:

1. **esbuild ≤ 0.24.2** — `Access-Control-Allow-Origin: *` on dev server lets any website fetch dev-server responses (GHSA-67mh-4wv8-2f99). Reaches us via `vite@5 → esbuild@0.21.5` and `vitest → vite → esbuild`.
2. **vite ≤ 5.4.21** — middleware mode dev-server filename leak (GHSA-4w7w-66w2-5vf9).

Both are dev-only. Production builds do not include vite/esbuild runtime. The attack surface is the local developer machine while running `pnpm dev` or `vitest --watch` and visiting a malicious site in the same browser.

## Decision

Waive both for Phase 0. Do not bump Vite to 6.x in this phase.

Reasons:
- Pre-condition for Phase 1 deliverables; bumping vite 5 → 6 risks breaking the existing web build (config/plugin compat) and is out-of-scope for Memory Cockpit work.
- Local-only impact; mitigated by not browsing untrusted sites while `pnpm dev`/`vitest --watch` is running.
- The fastify/static fix (which is the only one in production server code) was applied.

## Consequences

- Track as TASK-001: bump vite to 6.x in a dedicated PR after Phase 1 ships, with a regression test for the web dev/build flow.
- `pnpm audit` will continue to report 2 moderate vulns until the bump lands; this is expected.
